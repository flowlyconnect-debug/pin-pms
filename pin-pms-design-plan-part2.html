AButton-refaktori — 3 turvallista lähestymistapaa

Ymmärrettävää että alkuperäinen prompti rikkoi rakenteen — se yritti korvata KAIKKI napit kerralla. Tässä kolme turvallisempaa polkua. Valitse riskinsietokykysi mukaan, ja ne voi myös yhdistää (A+B+C peräkkäin).

A
Tapa A — CSS-only refresh (ei koske komponentteihin)
Pieni riski ~30 min

Tyyli kaikki <button>-elementit ja <a class="btn">-linkit globaaleilla CSS-säännöillä. Komponentit pysyvät täysin koskemattomina. Saa 80% visuaalisesta hyödystä 5% riskillä.

+ Hyvät puolet
  • Mikään ei rikkoonnu — vain CSS
  • Toimii heti kaikissa nykyisissä napeissa
  • Voi peruuttaa yhdellä git-revertillä
− Huonot puolet
  • Ei TypeScript-tukea variantteille
  • Loading-tila ja icon-prop puuttuvat
  • Tarvitsee data-attribuutteja varianttiin
PROMPT A · Button CSS-refresh (ei breaking) 
Tehtävä: Päivitä kaikkien nappien tyyli CSS-tasolla ILMAN että muutat
mitään komponentteja tai TSX-tiedostoja.

1. Luo src/styles/buttons.css ja importaa se globaalisti.

2. Käytä data-attribuutteja varianttiin — ÄLÄ uusia luokkia jotka voisivat
   törmätä olemassa oleviin:

  outline: none;

3. Lisää samat säännöt:
   - secondary: bg surface, border 1px var(--border-strong), text var(--text)
   - ghost: bg transparent, text var(--text-muted)
   - danger: bg #DC2626

4. Päivitä OLEMASSA OLEVAT button-tagit lisäämällä vain data-variant:
   - "Uusi lasku", "Suodata", "Toteuta", "Tallenna" → data-variant="primary"
   - "Merkitse myöhässä", "Peruuta", "Vaihda" → data-variant="secondary"
   - Pelkät tekstilinkit ("Näytä", "Muokkaa") → ÄLÄ vielä kosketa

5. Älä luo Button-komponenttia. Älä koske komponenttien JSX-rakenteeseen
   muuten kuin lisäämällä yksi data-attribuutti.

Tee yksi pull request ja varmista että kaikki nykyiset toiminnot
(submit, onClick, form-submission) toimii edelleen.

Pieni mini-graafi (60×20px) jokaisen ison luvun alla → näyttää 30 päivän trendin yhdellä silmäyksellä. Ei kirjastoa.

PROMPT · Sparkline-komponentti 
Tehtävä: Luo kevyt Sparkline-komponentti ilman kirjastoja.

1. Luo src/components/ui/Sparkline.tsx:

interface SparklineProps {
  data: number[]
  width?: number
  height?: number
  stroke?: string
  fill?: string
  showDots?: boolean
}

export function Sparkline({ data, width=60, height=20,
                           stroke='var(--primary)',
                           fill='var(--primary-soft)',
                           showDots=false }: SparklineProps) {
  if (!data || data.length < 2) return null
  const min = Math.min(...data)

    height - ((v - min) / range) * height
  ])

    `${i === 0 ? 'M' : 'L'} ${x.toFixed(1)} ${y.toFixed(1)}`).join(' ')
  const areaPath = `${linePath} L ${width} ${height} L 0 ${height} Z`

}

// Stable hash → hue (0-360)
function nameHash(name: string): number {
  let h = 0
  for (let i = 0; i < name.length; i++) {
    h = ((h << 5) - h) + name.charCodeAt(i)

  }
  return Math.abs(h) % 360
}
