0Designfilosofia

Hyvä SaaS-design ei ole "kaunis" — se on hiljainen. Se ei kilpaile käyttäjän kanssa. Se ohjaa silmän siihen mikä on tärkeää, ja kaikki muu jää taustalle. Tavoite: Pin PMS näyttää kalliilta, hallitulta ja modernilta — ei "tehty viikossa".

1. Vähemmän on enemmän

Poista turhat varjostukset, värit ja viivat. Jokainen pikseli pitää ansaita paikkansa. Jos jokin elementti ei lisää tietoa, se häiritsee.

2. Hierarkia ohjaa silmää

Tärkeä luku iso ja tumma, metadata pieni ja vaalea. Käyttäjä näkee 0,5 sekunnissa mitä tehdä seuraavaksi.

3. Yksi väri = yksi merkitys

Punainen = brändi & primaaritoiminto. Vihreä = onnistuminen. Keltainen = varoitus. Ei sekoiteta.

4. 8px-grid pyhä

Kaikki välistys, paddingit, marginit pyöristetään 4 tai 8 pikselin moninkertaisiin. Tämä on suurin yksittäinen ero amatööri- ja prodesignin välillä.

5Spacing — tärkein ero amatöörin ja pron välillä

10WOW-parannukset

Pieniä yksityiskohtia jotka erottavat Pin PMS:n geneerisistä admin-paneeleista. Nämä ovat "free" — eivät vaadi suurta refaktorointia mutta tuntuvat kalliilta.

🎬 Skeleton-loaders Korkea

Latauksen aikana näytä haalea harmaa "luuranko" sisällöstä — ei tyhjä valkoinen ruutu eikä spinner. Kuten Linear, Notion. Käytä CSS animation: pulse.

✨ Mikro-animaatiot Korkea

Hover, focus, modaali avautuu — kaikkialla 150–200ms ease transition. Ei yhtäkkiä, ei liian hitaasti. Tämä on suurin yksittäinen "kallis tuntu" -tekijä.

⌨️ Komentopaletti (Cmd+K) Korkea

Painamalla ⌘K aukeaa hakupaletti josta voi: hakea asiakasta, varausta, navigoida sivuille, luoda uuden laskun. Power-user-feature joka tekee vaikutuksen demoissa.

📊 Sparklines stat-korteissa Keski

Pieni mini-graafi (~60×20px) jokaisen ison luvun alla → näyttää 30 päivän trendin. SVG, ei kirjastoa.

🌓 Dark mode kunnolla Keski

Sinulla on jo "Auto" -valinta — tee siitä oikeasti hyvä. Ei pelkästään invertoitu, vaan oma paletti tummana (slate-950 bg, slate-100 text).

🔔 Toast-notifikaatiot Korkea

Kun käyttäjä tallentaa, poistaa, lähettää — näytä oikeasta yläkulmasta liukuva toast (esim. Sonner-tyylinen). Ei alert(), ei "Tallennettu!" page reload.

📆 Empty-states Korkea

"Ei laskuja" → näytä iso ystävällinen kuva, otsikko "Ei vielä laskuja", primary CTA "Luo ensimmäinen lasku". Ei tyhjä taulukko.

🎨 Avatarit värillä Matala

Asiakas-/käyttäjälistoissa: jos ei profiilikuvaa, näytä gradientti-pallo nimikirjaimilla. Hash nimestä → konsistentti väri. Pikkuyksityiskohta joka ilahduttaa.

📋 Kopioi-toiminnot Keski

Lasku-numerot, asiakas-ID:t, viitenumerot — kaikki klikattavissa "kopioi"-toiminnolla (näytä copy-icon hover-tilassa).

📈 Kalenteri-viewi parannus Korkea

Käyttöaste-kalenteri kuukausinäkymässä: värikoodatut "soluja" (vihreä = vapaa, sininen = varattu, harmaa = tyhjä). FullCalendar.io tyylinen.

11Lisäehdotukset (et maininnut näitä, mutta nämäkin pitäisi katsoa)

Asioita joita huomasin screenshoteistasi, mitkä eivät ole värissä/typografiassa, mutta hyödyttävät kokonaisuutta.

🇫🇮 Lukujen muotoilu Korkea

"500.00 EUR" → "500,00 €". "BIL-1-00000001" voisi olla "BIL-001". Suomalaiset päivät: "2026-05-15" → "15.5.2026". Intl.NumberFormat('fi-FI').

🏢 Brändi-logo Korkea

Punainen täyttölaatikko ei ole logo. Tee oikea wordmark — esim. "PinPMS" custom kirjasimella, tai geometrinen mark (P-kirjain talo-elementissä). Voin auttaa — pyydä erikseen.

📤 Empty drop-down "Organisaatio" Keski

"Organisaatio: Flowly · Vaihda" -rivi etusivulla on hieman irrallinen. Siirrä se sidebariin (käyttäjä-avatarin viereen) → vapauttaa tilaa.

🔍 Globaali haku Korkea

Topbarissa iso haku-kenttä joka hakee yli kaikki entiteetit (asiakkaat, kohteet, laskut, varaukset). Yhdistä Cmd+K -palettiin.

⚙️ Asetukset eri sivulle Matala

"Teema: Auto" -pudotus sidebarin alalaidassa on kömpelö. Siirrä asetukset omalle sivulle (käyttäjä-menun alle "Asetukset").

📊 Kalenteri konfliktit -näkymä Keski

"Kalenteriristiriidat" sidebarissa on outo terminologia. Vaihda muotoon "Konfliktit" (lyhyempi) ja näytä ne korostetusti dashboardissa kun niitä on.

📧 Sähköposti-templatet Keski

Laskut, sopimukset, muistutukset — varmista että sähköpostit, jotka PMS lähettää asiakkaille, ovat samoin viimeisteltyjä. Tämä on usein unohdettu pinta.

♿ Saavutettavuus (WCAG AA) Korkea

Kontrastisuhteet, focus-renkaat, näppäimistö-navigaatio, ARIA-labelit. EU:n saavutettavuusdirektiivi (2025) edellyttää tätä jo nyt.

📱 Mobiili-näkymä Keski

Sidebar collapse → bottom nav mobiilissa. Taulukot → korttinäkymä. Property managerit kävelevät kohteilla — tämä on iso plus.

🔐 Audit-loki paranneltu Matala

Sinulla on jo "Audit-loki" sidebarissa — tee siitä oikeasti hyvä. Aikajana, suodattimet käyttäjän/objektin/toiminnon mukaan.

Tehtävä: Luo design system -tokenit Pin PMS:lle.

1. Lisää Inter-fontti Google Fontsista applikaation root-templateen (preconnect + font-display: swap).
2. Luo tiedosto src/styles/tokens.css joka sisältää CSS custom properties:
   - Värit: --bg (#FAFAF9), --surface (#FFFFFF), --border (#E7E5E4), --border-strong (#D6D3D1), --text (#0C0A09), --text-muted (#57534E), --text-soft (#78716C), --primary (#B91C1C), --primary-hover (#991B1B), --primary-soft (#FEF2F2), --success (#15803D), --success-soft (#DCFCE7), --warning (#B45309), --warning-soft (#FEF3C7), --info (#1D4ED8), --info-soft (#DBEAFE), --sidebar (#0C0A09), --sidebar-text (#E7E5E4)

PROMPT 6 / 12 · Stat cards with trend & sparkline 
Tehtävä: Tee uusi StatCard-komponentti ja päivitä etusivun 4 isoa lukua.

Komponentti src/components/ui/StatCard.tsx, props:
- label (string)
- value (string | number) — formatoidaan automaattisesti euroilla / prosenteilla
- trend (number) — esim. +8.2 → "↗ +8,2% vs. ed. kk", negatiivinen punaisella alaspäin
- meta (string) — pieni alarivi
- icon (ReactNode) — pieni 28×28 ikoni-kapseli oikealla
- sparkline (number[]) — optional pieni svg-graafi pohjalla
- intent ('default' | 'success' | 'warning' | 'danger') — vaikuttaa ikonin taustaan

- 20px padding, 1px border var(--border), border-radius 10px, bg surface

- Meta: 12px text-muted

Sparkline: 60×20px SVG, polyline, stroke 1.5px primary, fill primary-soft 20% opacity.

- Käyttöaste (with sparkline last 30 days)

export const formatCurrency = (n: number) =>
  new Intl.NumberFormat('fi-FI', { style: 'currency', currency: 'EUR' }).format(n)
// → "1 240,00 €"

export const formatDate = (d: Date | string) =>
  new Intl.DateTimeFormat('fi-FI', { day: 'numeric', month: 'numeric', year: 'numeric' })
    .format(new Date(d))
// → "15.5.2026"

export const formatDateTime = (d: Date | string) =>
  new Intl.DateTimeFormat('fi-FI', { dateStyle: 'short', timeStyle: 'short' })
    .format(new Date(d))
// → "15.5.2026 14:30"

export const formatPercent = (n: number, decimals = 1) =>
  new Intl.NumberFormat('fi-FI', { style: 'percent', minimumFractionDigits: decimals, maximumFractionDigits: decimals })
    .format(n / 100)
// → "50,0 %"
